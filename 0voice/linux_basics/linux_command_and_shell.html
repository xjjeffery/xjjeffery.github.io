    cd(切换目录)¶

    cd 命令相当于我们在 Windows 中鼠标双击进入目录的操作，cd 的全称是 Change Directory，意思是交换目录。语法：

    cd [相对路径或绝对路径]

    使用案例：

    $ cd ~  # 表示回到自己的家目录
$ cd .. # 表示回到自己的上一级目录
$ cd -  # 表示回到上一次所在的目录

    pwd(显示当前目录)¶

    pwd 是 Print Working Directory 的缩写，也就是显示目前所在目录的命令。

    使用案例：

    $ pwd   # 单纯显示当前的目录
/home/jeffery/share/00_shell

# 显示出实际的工作目录，而非链接本身的目录名
$ cd /lib
$ pwd
/lib  # 列出当前的工作目录
$ pwd -P
/usr/lib # 列出实际的目录

$ ls -l /lib
lrwxrwxrwx 1 root root 7 Mar 14  2023 /lib -> usr/lib
# /lib 是一个符号链接，访问 /lib 目录就是在访问 /usr/lib 目录

    mkdir(创建一个新目录)¶

    在 Windows 中创建一个新目录只需在空白处右击新建一个即可，在 Linux 中使用 mkdir(make directory) 来创建一个新目录，其基本使用语法如下

    其中 -m 是配置文件的权限，不会依据默认权限(umask)来配置权限。-p 是允许在将所需要的目录递归创建

    使用案例：

    ~$ cd /tmp
tmp$ mkdir test # 创建一名为 test 的新目录
tmp$ mkdir test1/test2/test3/test4
mkdir: cannot create directory ‘test1/test2/test3/test4’: No such file or directory
# No such file or directory <== 没办法直接创建此目录啊！
tmp$ mkdir -p test1/test2/test3/test4 # 加了 -P 就可以自行递归创建多层目录

/tmp$ mkdir -m 711 test2
/tmp$ ls -l
drwxrwxr-x 3 0voice 0voice 4096 Sep 11 12:50 test
drwxrwxr-x 3 0voice 0voice 4096 Sep 11 12:53 test1
drwx--x--x 2 0voice 0voice 4096 Sep 11 12:54 test2

    上面的权限部分，如果没有加上 -m 来强制配置属性，系统会使用默认属性。如果我们使用 -m，如上例我们给予 -m 711 来给予新的目录 drwx--x--x 的权限。

    rmdir(删除一个空目录)¶

    在 Windows 中删除一个目录右击删除即可，而在 Linux 中需要使用 rmdir 命令删除目录，并且需要删除的目录必须是一个空目录。

    加上 -p 选项表示连同上一级的空目录也一起删除，使用案例：

    /tmp$ ls -l # 看看有多少目录存在？
drwxrwxr-x 3 0voice 0voice 4096 Sep 11 12:50 test
drwxrwxr-x 3 0voice 0voice 4096 Sep 11 12:53 test1
drwx--x--x 2 0voice 0voice 4096 Sep 11 12:54 test2
/tmp$ rmdir test  # 可直接删除掉，没问题
/tmp$ rmdir test1 # 因为尚有内容，所以无法删除！
rmdir: ’test1‘: Directory not empty
/tmp$ rmdir -p test1/test2/test3/test4
/tmp$ ls -l # 您看看，底下的输出中 test 与 test1 不见了！
drwx--x--x 2 0voice 0voice 4096 Sep 11 12:54 test2

    如果需要删除非空目录，则要使用 rm 命令。

    cp(复制文件或目录)¶

    cp 拷贝文件和目录的语法如下

    其中：

    • -a：相当于 -pdr 的意思，至于 pdr 请参考下列说明(常用)
    • -d：若来源文件为链接文件的属性(link file)，则复制链接文件属性而非文件本身
    • -f：为强制(force)的意思，若目标文件已经存在且无法开启，则移除后再尝试一次
    • -i：若目标文件(destination)已经存在时，在覆盖时会先询问动作的进行(常用)
    • -l：进行硬式链接(hard link)的链接文件创建，而非复制文件本身
    • -p：连同文件的属性一起复制过去，而非使用默认属性(备份常用)
    • -r：递归持续复制，用于目录的复制行为(常用)
    • -s：复制成为符号链接(symbolic link)
    • -u：若 destination 比 source 旧才升级 destination

    用 jeffery 身份，将 jeffery 目录下的 .bashrc 复制到 /tmp 下，并命名为 bashrc。

    ~$ cp ~/.bashrc /tmp/bashrc
~$ cp -i ~/.bashrc /tmp/bashrc
cp: overwrite ’/tmp/bashrc‘? n # n 不覆盖，y 为覆盖

    rm(移除文件或目录)¶

    在 Windows 中删除文件和目录的操作都是相同的，但是在 Linux 中删除文件使用 rm，此命令也可用于删除目录。

    常用的选项与参数：

    • -f：就是 force 的意思，忽略不存在的文件，不会出现警告信息
    • -i：互动模式，在删除前会询问使用者是否动作
    • -r：递归删除啊！最常用在目录的删除了！这是非常危险的选项！

    使用案例：将刚刚在 cp 的实例中创建的 bashrc 删除掉

    ~$ rm -i bashrc
rm: remove regular file `bashrc'? y

    如果加上 -i 的选项就会主动询问，避免你删除到错误的文件。

    mv(移动文件与目录/修改名称)¶

    在 Windows 中剪切文件也是通过右击鼠标来操作，在 Linux 中使用 mv 命令。

    常用的选项与参数：

    • -f：force 强制的意思，如果目标文件已经存在，不会询问而直接覆盖
    • -i：若目标文件(destination)已经存在时，就会询问是否覆盖
    • -u：若目标文件已经存在，且 source 比较新，才会覆盖

    使用案例：复制一文件，创建一目录，将文件移动到目录中

    ~$ cd /tmp
/tmp$ cp ~/.bashrc bashrc
/tmp$ mkdir mvtest
/tmp$ mv bashrc mvtest

    Linux 文件内容查看¶

    cat(文本输出)¶

    cat 是将文件内容从第一行开始一次全部显示出来，使用语法如下

    常用选项和参数：

    • -A：相当于 -vET 的整合选项，可列出一些特殊字符而不是空白而已
    • -b：列出行号，仅针对非空白行做行号显示，空白行不标行号
    • -E：将结尾的断行字节 $ 显示出来

# Magic local data for file(1) command.
# Insert here your local magic data. Format is described in magic(5).

    tac(与 cat 相反)¶

    tac 与 cat 的命令刚好相反，文件内容是从最后一行开始显示，使用语法如下

    使用案例：

    ~$ cat /etc/magic
# Insert here your local magic data. Format is described in magic(5).
# Magic local data for file(1) command.

    nl(显示行号)¶

    此命令可以带有行号的方式显示文件数据，使用语法如下

    常用的选项与参数：

    • -b：指定行号指定的方式，主要有两种
    • -b a：表示不论是否为空行，也同样列出行号(类似 cat -n)
    • -b t：如果有空行，空的那一行不要列出行号(默认值)
    • -n：列出行号表示的方法，主要有三种
    • -n ln：行号在荧幕的最左方显示
    • -n rn：行号在自己栏位的最右方显示，且不加 0
    • -n rz：行号在自己栏位的最右方显示，且加 0
    • -w：行号栏位的占用的位数

    使用案例：

    ~$ nl /etc/magic
     1  # Magic local data for file(1) command.
     2  # Insert here your local magic data. Format is described in magic(5).

    more(一页一页翻动)¶

    上面的命令都是一次将文件的所有内容输出，对于内容较多的文件，这种方式可能会影响我们浏览文件内容。使用 more 命令可以从第一行开始显示部分文件内容，然后逐页逐页的浏览，使用语法如下：

    使用案例：

    ~$ more ~/.bashrc
# ~/.bashrc: executed by bash(1) for non-login shells.
# see /usr/share/doc/bash/examples/startup-files (in the package bash-doc)
# for examples

# If not running interactively, don't do anything
case $- in
    *i*) ;;
      *) return;;
esac

# don't put duplicate lines or lines starting with space in the history.
# See bash(1) for more options
HISTCONTROL=ignoreboth

# append to the history file, don't overwrite it
shopt -s histappend

# for setting history length see HISTSIZE and HISTFILESIZE in bash(1)
HISTSIZE=1000
HISTFILESIZE=2000

# check the window size after each command and, if necessary,
# update the values of LINES and COLUMNS.
shopt -s checkwinsize
--More--(17%)

    在 more 这个程序的运行过程中，常用的按键有：

    • 空白键(space)：代表向下翻一页
    • Enter：代表向下翻『一行』
    • /字串：代表在这个显示的内容当中，向下搜寻『字串』这个关键字
    • :f：立刻显示出文件名以及目前显示的行数
    • q：代表立刻离开 more，不再显示该文件内容
    • b 或 ctrl+b：代表往回翻页，不过这动作只对文件有用

    less¶

    less 命令比 more 更加的好用，支持很多 vim 的按键方式，有关 less 的使用语法请自行查阅 man 手册。

    进入 less 运行时的程序可用以下按键：

    • 空白键：向下翻动一页
    • pagedown：向下翻动一页
    • pageup：向上翻动一页
    • /字串：向下搜寻『字串』的功能
    • ?字串：向上搜寻『字串』的功能
    • n：重复前一个搜寻 (与 / 或 ? 有关)
    • N：反向的重复前一个搜寻 (与 / 或 ? 有关)
    • q：离开 less 这个程序

    Shell 编程¶

    更多的 shell 教程，请阅读 Bash 脚本教程和《鸟哥的 Linux 私房菜》第三部分。

    Shell 是一个用 C 语言编写的程序，是一个命令解释器，是用户使用 Linux 的桥梁。而 Shell 脚本(shell script)是一种为 shell 编写的程序。常见 shell 有以下几种：

    • Bourne Shell（/usr/bin/sh 或/bin/sh）
    • Bourne Again Shell（/bin/bash）
    • C Shell（/usr/bin/csh）
    • K Shell（/usr/bin/ksh）

    但是在诸多的 Linux 系统发行版中，Bash 是最常见，也是使用最多并且是大多数 Linux 系统默认的 Shell。

    shell 脚本的基本形式：

    1. shell 脚本的扩展名均为 .sh
    2. shell 脚本中的第一行必须是 #! <路径>，来告诉系统需要什么解释器来执行
    3. shell 脚本必须让该文件拥有可执行权限或者让该文件作为解释器的参数，执行命令

    Shell 变量¶

    shell 脚本中也可以使用变量来存储值，变量的定义有以下几个规则：

    • 变量名和等号之间不能有空格
    • 命名只能使用英文字母，数字和下划线，首个字符不能以数字开头
    • 名称中间不能有空格
    • 不能使用标点符号
    • 不能使用 bash 里的关键字(可用 help 命令查看保留关键字)

    例如：

    # 有效的 shell 变量
zerovoice
ZERO_VOICE
_ZERO_VOICE
Zerovoice0

# 无效的 shell 变量
0voice
?voice
zero*voice

    使用变量¶

    使用一个定义过的变量，只要在变量名前加上 $ 符号即可，如：

    domain="www.github.com"
echo $domain
echo ${domain}

    变量外的大括号，$domain 和 ${domain} 效果一样，只是后者可以帮助解释器识别变量的边界，如

    for skill in C CPP Linux Shell; do
  echo "I am good at ${skill}Code"
done

    如果不给 skill 变量加上大括号，写成了 echo "I am good at ${skill}Code"，解释器就会把 $skillCode 当成一个变量。

    使用 readonly 命令可以将变量定义为只读变量，只读变量的值不能被改变。下面的例子尝试改变只读变量，结果报错：

    #! /bin/bash

url="http://www.google.com"
readonly url

    删除变量¶

    使用 unset 命令可以删除变量，语法：

    unset variable_name

    变量被删除后不能再次使用，unset 不能删除只读变量。

    使用示例：

    #!/bin/bash

url="http://www.baidu.com"
unset url
echo $url

    执行上述脚本，没有任何输出。

    变量类型¶

    运行 shell 时，会同时存在三种变量：

    1. 局部变量：局部变量在脚本或命令中定义，仅在当前 shell 脚本实例中有效，其他 shell 启动的程序不能访问局部变量
    2. 环境变量(全局变量)：所有的程序，包括 shell 启动的程序，都能访问环境变量，有些程序需要环境变量保证其正常运行。必要的时候 shell 脚本也可以定义环境变量
    3. shell 变量：shell 变量是由 shell 程序设置的特殊变量。shell 变量中有一部分是环境变量，有一部分是局部变量，这些变量保证了 shell 的正常运行

    Shell 字符串¶

    字符串是 shell 编程中最常用最有用的数据类型（除了数字和字符串，也没啥其它类型好用了），字符串可以用单引号，也可以用双引号，也可以不用引号。

    单引号¶

    单引号字符串的使用案例：

    #! /bin/bash

name=jeffery
str='my name is $name'
echo $str

    运行上述脚本，输出结果：

    $ chmod u+x 03_string.sh
$ ./03_string
my name is $name

    为什么输出结果不是我们所想的，是因为单引号字符串有几个限制：

    • 单引号里的任何字符都会原样输出，单引号字符串中的变量都是无效的
    • 单引号字串中不能出现单独一个的单引号(对单引号使用转义字符也不行)，但可成对出现，作为字符串拼接使用

    双引号¶

    上面脚本的字符串使用双引号，会发生什么

    #! /bin/bash

name="jeffery"
str='my name is $name'
echo $str

    运行上述脚本，输出结果：

    $ chmod u+x 03_string.sh
$ ./03_string
my name is jeffery

    双引号的基本使用与单引号相同，但是双引号里可以有变量、可以有转义字符。

    拼接字符串¶

    使用案例：

    # 拼接字符串
your_name="0voice"
# 使用双引号拼接
greeting="hello, "$your_name" !"
greeting_1="hello, ${your_name} !"
echo $greeting $greeting_1

#使用单引号拼接
greeting_2='hello, '$your_name' !'
greeting_3='hello, ${your_name} !'
echo $greeting_2 $greeting_3

    输出结果

    hello, 0voice ! hello, 0voice !
hello, 0voice ! hello, ${your_name} !

    获取字符长度¶

    使用案例：

    string="abcd"
echo ${#string} # 输出 4

    提取子字符串¶

    使用案例：从字符串第 2 个字符开始截取 4 个字符

    string="0voice is a great college"
echo ${string:1:4}

    查找字符位置¶

    使用案例：查找字符 i 或 o 的位置

    string="0voice is great college"
echo `expr index "$string" io` # 输出 3

    Shell 数组¶

    bash 支持一维数组（不支持多维数组），并且没有限定数组的大小。类似于 C 语言，数组元素的下标由 0 开始编号。获取数组中的元素要利用下标，下标可以 是整数或算术表达式，其值应大于或等于 0。

    定义数组¶

    在 Shell 中，用括号来表示数组，数组元素用“空格”符号分割开。定义数组的一般形式为：

    数组名=(值1 值2 值3 ... 值n)

    使用案例：

    # 方法一
nums=(1 34 12 3)

# 方法二：单独定义数组的各个分量
nums[0]=1
nums[1]=12
nums[n]=132

    单独定义数组时，可以不使用连续的下标，而且下标的范围没有限制。

    读取数组¶

    读取数组元素值的一般格式是：${数组名[下标]}，如下所示：

    nums=(1 34 12 3)
value=${nums[2]}

    使用 @ 符号可以获取数组中的所有元素，例如

    nums=(1 34 12 3)
echo ${nums[@]}

    获取数组的长度¶

    获取数组长度的方法与获取字符串长度的方法相同 ${#数组名[@]}，例如：

    # 获取数组的长度
echo ${#nums[@]}
# 获取单个元素的长度
echo ${#nums[2]}

    Shell 注释¶

    以 # 开头的行就是注释，会被解释器忽略。通过每一行加一个 # 号设置多行注释，像这样：

    #-------------------------------------------
# slogan：一切只为渴望更优秀的你
#--------------------------------------------
##### 用户配置区 开始 #####-
#
#
# 这里可以添加脚本描述信息

    如果在开发过程中，遇到大段的代码需要临时注释起来，过一会儿又取消注释，怎么办呢？每一行加个 # 符号太费力了，可以把这一段要注释的代码用一对花括号括起来，定义成一个函数，没有地方调用这个函数，这块代码就不会执行，达到了和注释一样的效果。

    Shell 传递参数¶

    我们可以在执行 Shell 脚本时，向脚本传递参数，脚本内获取参数的格式为：$n。n 代表一个数字，1 为执行脚本的第一个参数，2 为执行脚本的第二个参数，以此类推 ……

    以下实例我们向脚本传递三个参数，并分别输出，其中 $0 为执行的文件名：

    #! /bin/bash

echo "Shell 传递参数实例";
echo "执行的文件名: $0";
echo "第一个参数: $1"
echo "第一个参数: $2"
echo "第一个参数: $3"

    运行此脚本，输出结果如下：

    $ chmod u+x test.sh
$ ./test.sh 1 2 3
Shell 传递参数实例
执行的文件名：./test.sh
第一个参数为：1
第二个参数为：2
第三个参数为：3

    特殊字符¶

    除了使用上述的方式获取参数，还有其他特殊的字符来处理参数

    使用案例：

    #! /bin/bash

echo "Shell 传递参数实例";
echo "第一个参数: $1";
echo "参数个数: $#";
echo "传递的参数作为一个字符串表示: $*"

    运行此脚本，输出结果如下：

    $ chmod u+x test.sh
$ ./test.sh 1 2 3
Shell 传递参数实例
执行的文件名：./test.sh
第一个参数为：1
参数个数为：3
传递的参数作为一个字符串显示：1 2 3

    $* 与 $@ 的区别¶

    • 相同点：都是引用所有参数。
    • 不同点：只有在双引号中体现出来，假设在脚本运行时写入三个参数 1、2、3，则 * 等价于 "1 2 3"(当作一个整体)，而 @ 等价于 "1" "2" "3"(每个都是单独的一个)。
    #! /bin/bash

echo "-- \$* 演示 ---"
for i in "$*";do
  echo $i
done

echo "-- \$@ 演示 ---"
for i in "$@";do
  echo $i
done

    执行脚本，输出结果如下所示：

    $ chmod +x test.sh
$ ./test.sh 1 2 3
-- $* 演示 ---
1 2 3
-- $@ 演示 ---
1
2
3
